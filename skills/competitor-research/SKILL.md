# 競合調査スキル (Competitor Research)

## 目的

`config/settings.yaml` の `research.keywords` に定義されたキーワードでXを検索し、
人気投稿のパターンを分析して `output/research/YYYY-MM-DD.md` に保存する。

## 実行手順

### 1. 設定読み込み
```bash
cat config/settings.yaml
```
以下の値を取得:
- `research.keywords` (検索キーワード)
- `research.min_likes` (最低いいね数)
- `research.lookback_hours` (対象時間範囲)
- `research.sample_size` (収集件数)
- `theme.genre` (テーマ整合性チェック用)

### 2. X検索の実行

各キーワードについて、以下のURLパターンでX検索を行う:
```
https://x.com/search?q={キーワード}%20min_faves%3A{min_likes}&f=live&src=typed_query
```

WebFetchで取得できない場合は、以下の代替手段を順に試す:
1. `https://nitter.net/search?q={キーワード}&f=tweets` (Nitterミラー)
2. SearchAPIなどのMCPコネクタが接続されていればそれを使用
3. 失敗時は`output/research/YYYY-MM-DD-error.md`にエラーログを残し、過去の調査結果(直近3日分)を参照する

### 3. 投稿の抽出・分析

取得した投稿から以下を抽出:
- **投稿本文**(全文)
- **エンゲージメント**(いいね/RT/返信数)
- **フック**(冒頭1〜2行)
- **構造パターン**(箇条書き/ストーリー/問いかけ/逆張り など)
- **使われている数字や具体例**

### 4. パターン分析

`sample_size`件を集計し、以下を導出:

#### A. バズりやすいフック型(上位5種)
例:
- 「〜という人へ」型(共感呼びかけ)
- 「〇〇したら△△だった」型(変化提示)
- 「知らないと損する〇〇」型(損失回避)
- 「実は〜」型(意外性)
- 数字提示型「3日で〇〇した」

#### B. 頻出トピック・キーワード
出現回数の多い名詞・固有名詞をリスト化

#### C. 文章構造の傾向
- 平均文字数
- 改行パターン(1文1行/段落型)
- 絵文字の有無・頻度

### 5. 出力

`output/research/YYYY-MM-DD.md` を以下のフォーマットで作成:

```markdown
# 競合調査レポート YYYY-MM-DD

## サマリー
- 調査キーワード: [一覧]
- 収集件数: X件
- 平均エンゲージメント: いいね X / RT X

## バズりやすいフック型 TOP5
1. [型名] - 例: 「具体例の引用」
2. ...

## 頻出トピック
- [キーワード1] (X回)
- [キーワード2] (X回)

## 文章構造の傾向
- 平均文字数: X文字
- 改行パターン: [傾向]
- 絵文字使用率: X%

## 投稿サンプル(エンゲージメント上位5件)
### 1. いいね数: X
[投稿本文]
**フック型**: ...
**学べる要素**: ...

[以下、上位5件まで]

## viral-post-writerへの引き継ぎメモ
- 今日推奨するフック型: [上位2つ]
- 避けるべき表現: [飽和している言い回し]
- 取り入れたいトピック: [伸びている話題]
```

## エラーハンドリング

- X側のレート制限/取得失敗 → 直近3日の過去レポートを参照させる
- 投稿が0件 → キーワードを広げる提案をログに残す
- `theme.ng_words`に該当する投稿は分析対象から除外

## 完了条件

`output/research/YYYY-MM-DD.md` が作成され、最低1つのフック型と5件以上の投稿サンプルが記載されていること。

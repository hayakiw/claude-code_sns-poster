# スプレッドシート記録スキル (Sheet Logger)

## 目的

`viral-post-writer`スキルが生成した投稿案を、`config/settings.yaml`で指定された
Googleスプレッドシートに行追加する。

## 前提

- Routinesに **Google Drive コネクタ** または **Google Sheets MCPコネクタ** が接続されていること
- `config/settings.yaml` の `spreadsheet.url` に有効なスプレッドシートURLが設定されていること
- `output/posts/YYYY-MM-DD.md` が生成済みであること

## 実行手順

### 1. 設定とインプット読み込み

```bash
cat config/settings.yaml
cat output/posts/$(date +%Y-%m-%d).md
```

取得する値:
- `spreadsheet.url` → スプレッドシートID抽出
- `spreadsheet.sheet_name` → 対象シート名
- `generation.schedule_times` → 投稿予定時刻リスト
- 投稿案3件の本文

### 2. スプレッドシートIDの抽出

URLからIDを抽出する:
```
https://docs.google.com/spreadsheets/d/{この部分がID}/edit
```

### 3. ヘッダー行の確認(初回のみ)

シートが空の場合、1行目に以下のヘッダーを書き込む:
```
A: 投稿予定日 | B: 投稿予定時間 | C: プラットフォーム | D: 投稿テキスト案 | E: 画像・動画リンク | F: ステータス
```

### 4. 投稿予定日時の決定

- **投稿予定日**: 生成日 + 1日 (翌日)
  ```bash
  date -d "tomorrow" +%Y-%m-%d
  ```
- **投稿予定時間**: `generation.schedule_times`の順に割り当て
  - 案1 → 07:00
  - 案2 → 12:00
  - 案3 → 19:00

### 5. 行追加の実行

利用可能なコネクタに応じて以下のいずれかを実行:

**Google Drive コネクタの場合:**
スプレッドシートをDriveから開いて、行追加APIで書き込む。

**Google Sheets MCPの場合:**
`spreadsheet_id` と `range` を指定して `append` する。

3件分を3行として追記する。各行の内容:

| 列 | 値 |
|---|---|
| 投稿予定日 | YYYY-MM-DD (翌日) |
| 投稿予定時間 | 07:00 / 12:00 / 19:00 |
| プラットフォーム | Threads |
| 投稿テキスト案 | 投稿本文(改行を含むそのまま) |
| 画像・動画リンク | (空欄) |
| ステータス | 未着手 |

### 6. 書き込み確認

書き込み後、書き込んだ行数とスプレッドシートURLをログに残す:
```
✅ 3件を以下に記録しました
   {spreadsheet.url}
   投稿予定日: YYYY-MM-DD
   時刻: 07:00 / 12:00 / 19:00
```

## エラーハンドリング

| エラー | 対処 |
|---|---|
| URLからIDが抽出できない | 設定ファイルのURL形式を確認するようログ出力 |
| コネクタがない/権限エラー | エラー内容と再認証手順をログに残し、`output/posts/`の内容は保持 |
| 既に同日の行がある | 重複させず、上書きするか追加するかを設定で制御(デフォルト: 追加) |
| シート名が存在しない | `spreadsheet.sheet_name`の値を再確認するようログ出力 |

## データ整形の注意

- 投稿テキストの **改行を保持する** (セル内改行として書き込む)
- 文字数500を超える場合でも切り詰めない(Threadsの仕様変更に備える)
- ダブルクォート/カンマを含むテキストでも崩れない形式で書き込む

## 完了条件

スプレッドシートに3行が追加され、各行の **ステータス列が「未着手」** になっていること。
書き込んだ行数と対象URLが標準出力に表示されていること。

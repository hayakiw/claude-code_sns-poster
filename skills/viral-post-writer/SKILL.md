# バズる投稿文作成スキル (Viral Post Writer)

## 目的

`competitor-research`スキルが作成したレポートと`config/settings.yaml`を入力に、
Threads向けの投稿文を **3案** 生成し、`output/posts/YYYY-MM-DD.md` に保存する。

## 前提

このスキルを呼ぶ前に、`competitor-research`スキルが当日分のレポート(`output/research/YYYY-MM-DD.md`)を作成済みであること。
未作成の場合は、最新のレポートを使用する。

## 実行手順

### 1. インプット読み込み

```bash
cat config/settings.yaml
cat output/research/$(date +%Y-%m-%d).md  # 当日分。なければ最新
```

設定から取得:
- `theme.genre` / `target_audience` / `tone` / `persona` / `ng_words`
- `generation.posts_per_day` (生成件数)
- `generation.max_chars` (文字数上限)

### 2. バズる投稿の構造ルール

以下の **3層構造** を必ず守る:

#### 第1層: フック (1〜2行 / 60文字以内)
- 競合調査で抽出された「上位2つのフック型」を採用
- 3案でフック型を **被らせない**(例: 1案目=共感呼びかけ、2案目=数字提示、3案目=逆張り)
- スクロールを止めるための具体性 or 違和感を入れる

#### 第2層: 本文 (3〜6行)
- 結論ファースト
- 1文1改行 or 短い段落
- 具体例・数字・固有名詞を最低1つ含める
- 抽象論で終わらない

#### 第3層: 締め (1〜2行)
- 行動喚起 or 共感の問いかけ
- 「保存推奨」「リプで教えて」などの導線
- ハッシュタグは使わない(Threadsでは効果が薄い)

### 3. 3案の役割分担

3案それぞれに **異なる目的** を持たせる:

| 案 | 目的 | フック型の例 |
|---|---|---|
| 案1 | リーチ拡大(共感型) | 「〜な人いませんか？」 |
| 案2 | 保存・シェア狙い(ノウハウ型) | 「3つの〇〇テクニック」 |
| 案3 | 議論喚起(逆張り/問題提起型) | 「実は〇〇は不要です」 |

### 4. 生成時の品質チェック

各案について以下を満たすこと:

- [ ] 文字数が `max_chars` 以内
- [ ] `theme.ng_words` を含まない
- [ ] `theme.genre` のテーマから外れていない
- [ ] `theme.tone` の文体になっている
- [ ] フックに固有名詞 or 数字 or 違和感がある
- [ ] 締めに次のアクションがある
- [ ] ハッシュタグ・URL羅列をしていない

満たさない場合は、満たすまで書き直す。

### 5. 出力

`output/posts/YYYY-MM-DD.md` を以下のフォーマットで作成:

```markdown
# 投稿案 YYYY-MM-DD

## 案1: 共感型(リーチ重視)
**フック型**: [型名]
**狙い**: [この案で何を達成したいか]
**文字数**: XXX / 500

---
[投稿本文(そのままコピペできる形)]
---

## 案2: ノウハウ型(保存重視)
**フック型**: [型名]
**狙い**: ...
**文字数**: XXX / 500

---
[投稿本文]
---

## 案3: 逆張り型(議論喚起)
**フック型**: [型名]
**狙い**: ...
**文字数**: XXX / 500

---
[投稿本文]
---

## sheet-loggerへの引き継ぎ
- 投稿予定日: YYYY-MM-DD (生成日翌日)
- 投稿予定時刻: 07:00 / 12:00 / 19:00 (案1→案2→案3の順)
- ステータス初期値: 未着手
```

## 禁止事項

- 競合投稿の **コピペや言い換えに留まる文章** を作らないこと(構造を学んでテーマを変える)
- `ng_words`に該当する表現
- 根拠のない断定(「絶対」「100%」など)
- 実在の個人や企業を中傷する表現
- 投稿者を特定するような個人情報

## 完了条件

`output/posts/YYYY-MM-DD.md` に3案が記載され、すべてが品質チェックを通過していること。
